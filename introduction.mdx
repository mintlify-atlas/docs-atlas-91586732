Camaleon CMS is a free, open-source content management system built as a Ruby on Rails engine — a flexible alternative to WordPress designed specifically for Rails developers. It lets you define custom content types and custom fields without writing any code, manage multiple sites from a single installation, and extend everything through themes and plugins.

## Key features

- **Custom content types** — Create structured content without code using post types, categories, and tags
- **Custom fields** — Text, textarea, image, gallery, checkbox, radio, select, date, and more
- **Plugin system** — Hook-based architecture lets plugins modify any part of the CMS lifecycle
- **Theme system** — Full layout and template control with helper methods for assets and views
- **User roles** — Granular permissions with CanCanCan, supporting admin, editor, and custom roles
- **Media manager** — Upload files locally or to Amazon S3, with automatic image thumbnailing
- **SEO tools** — Sitemap generation, meta tags, robots.txt, and RSS feeds built in
- **Shortcodes and widgets** — Reusable content blocks and sidebar components
- **WordPress migration** — Import content from an existing WordPress installation

## Requirements

| Requirement | Version |
|-------------|---------|
| Ruby | 3.0+ |
| Ruby on Rails | 6.1+ |
| Database | PostgreSQL, MySQL 5+, or SQLite |
| ImageMagick | Any current version |

<Note>
  Camaleon CMS is distributed as a Ruby gem and integrates into any existing or new Rails application as a Rails Engine.
</Note>

## Get started

<Steps>
  <Step title="Add the gem">
    Add `gem "camaleon_cms"` to your Rails application's Gemfile.
  </Step>
  <Step title="Run the installer">
    Run `rails generate camaleon_cms:install` and `rake camaleon_cms:generate_migrations && rake db:migrate`.
  </Step>
  <Step title="Start your server">
    Run `rails server` and visit `http://localhost:3000` to complete setup.
  </Step>
</Steps>

See the [Quick Start guide](/quickstart) for the full step-by-step installation walkthrough.
