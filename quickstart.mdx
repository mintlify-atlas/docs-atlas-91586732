---

Camaleon CMS is distributed as a Ruby gem and mounts into any new or existing Rails application as a Rails Engine. The steps below walk you through a fresh installation — from creating a Rails app to completing the setup wizard in your browser.

## Requirements

| Requirement | Version |
|---|---|
| Ruby | 3.0+ |
| Ruby on Rails | 6.1+ |
| Database | PostgreSQL, MySQL 5+, or SQLite |
| ImageMagick | Any current version |

<Note>
  If you want to run your Rails app and its dependencies in Docker containers, follow the [Rails app containerization guide](https://github.com/owen2345/rails_app_containerization) before continuing.
</Note>

## Installation

<Steps>
  <Step title="Create a Rails app">
    If you already have a Rails application, skip to the next step.

    ```bash
    rails new my_project
    cd my_project
    ```
  </Step>

  <Step title="Add the gem to your Gemfile">
    Open your `Gemfile` and add the Camaleon CMS gem:

    ```ruby
    gem "camaleon_cms"
    ```

    If you want to run the latest development version directly from GitHub, use:

    ```ruby
    gem "camaleon_cms", github: "owen2345/camaleon-cms"
    ```
  </Step>

  <Step title="Install gem dependencies">
    ```bash
    bundle install
    ```

    This installs Camaleon CMS and all of its dependencies. The installer also appends a `PluginRoutes.draw_gems` call to your `Gemfile` so that gems declared by installed plugins and themes are picked up automatically.
  </Step>

  <Step title="Run the Camaleon CMS installer">
    ```bash
    rails generate camaleon_cms:install
    ```

    The generator does the following:

    - Copies `config/system.json` into your app with sensible defaults
    - Copies `lib/plugin_routes.rb` into your app
    - Creates the `app/apps/` directory structure for themes and plugins
    - Copies the built-in themes (`default`, `camaleon_first`, `new`) into `app/apps/themes/`
    - Updates `app/assets/config/manifest.js` to include the CMS asset links (Sprockets 4+)
    - Appends a `PluginRoutes.draw_gems` call to your `Gemfile`
  </Step>

  <Step title="Configure system.json (optional)">
    The installer places a `config/system.json` file in your Rails app root. You can edit it now to change settings such as the admin URL path, database table prefix, or available languages before running migrations.

    <Tip>
      This step is optional — the defaults work out of the box. See the [configuration reference](/configuration) for a full description of every available option.
    </Tip>

    ```json
    {
      "db_prefix": "cama_",
      "admin_path_name": "admin",
      "default_user_role": "client",
      "users_share_sites": true,
      "relative_url_root": "",
      "auto_include_migrations": false,
      "hooks": {}
    }
    ```
  </Step>

  <Step title="Generate and run migrations">
    Copy the Camaleon CMS migrations into your application, then apply them:

    ```bash
    rake camaleon_cms:generate_migrations
    rake db:migrate
    ```

    <Note>
      You can inspect the copied migration files in `db/migrate/` before running `rake db:migrate` if you need to customise them.
    </Note>
  </Step>

  <Step title="Start the server">
    ```bash
    rails server
    ```
  </Step>

  <Step title="Complete the setup wizard">
    Open [http://localhost:3000](http://localhost:3000) in your browser. Camaleon CMS will redirect you to the setup wizard on the first run, where you will:

    1. Create the first administrator account
    2. Set your site name and default language
    3. Choose a starting theme

    Once the wizard is complete your site is live and the admin panel is available at `http://localhost:3000/admin` (or whatever `admin_path_name` you configured).
  </Step>
</Steps>

## Deploy options

- **Heroku** — Deploy the [Camaleon CMS Sample App](https://github.com/brian-kephart/Camaleon-CMS-Sample) directly to Heroku with the one-click deploy button in that repository.
- **Docker** — Follow the [Rails app containerization guide](https://github.com/owen2345/rails_app_containerization) to run Camaleon CMS in a containerized environment.
